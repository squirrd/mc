  <action>
Replace all 74 print() statements with structured logging. For each file:

1. **Add import and create module-level logger** at top of file (after imports section):
   ```python
   import logging

   logger = logging.getLogger(__name__)
   ```

2. **Replace print() statements** following these level guidelines from CONTEXT.md:

   **INFO level** - Major operations users should know about:
   - "Downloading attachments for case number: {case_number}"
   - "Creating workspace for case {case_number}"
   - "Access token cached until {expiry_time}"
   - "Attachment downloaded: {filename}"

   **DEBUG level** - Detailed operation info for troubleshooting:
   - "Fetching case details from API"
   - "Checking workspace status"
   - "Token cache hit/miss"
   - "HTTP request details" (when --debug is active)

   **WARNING level** - Non-fatal issues:
   - "WARN: Large file download (>3GB)"
   - "Workspace file missing: {filename}"
   - Case status warnings

   **ERROR level** - Fatal issues requiring user action:
   - "FATAL: Failed file check"
   - "The env variable 'RH_API_OFFLINE_TOKEN' must be set"
   - Authentication failures
   - API errors

3. **Use lazy % formatting** (not f-strings) for performance:
   - Good: `logger.info('Downloading case %s', case_number)`
   - Bad: `logger.info(f'Downloading case {case_number}')`
   - See RESEARCH.md Pitfall 2 - lazy evaluation prevents expensive formatting when log level is disabled

4. **Preserve formatting for structured output** that users expect to parse:
   - LDAP card output in other.py (keep print() for now - this is intentional user output, not logging)
   - Workspace check output (status prefixes "OK", "WARN", "FATAL") - use logger.info() but maintain format
   - Separator lines `print("-" * 40)` -> `logger.info("-" * 40)` or remove if purely visual

5. **Keep exit(1) calls** - these are control flow, not logging

6. **Special cases:**
   - wizard.py prompts: Keep print() for interactive prompts, replace status messages with logger.info()
   - main.py: Replace print() for errors with logger.error(), keep exit(1)
   - pprint() calls: Replace with logger.debug() and format as JSON or structured text

Follow RESEARCH.md "Module-Level Logger Creation" pattern exactly.
