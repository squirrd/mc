
```python

# AFTER
logger.debug("Created directory: %s", path)
```

**Why this matters (from RESEARCH.md Pitfall 2):**
- F-strings: String is formatted BEFORE the logger checks if DEBUG level is enabled
- Lazy %: String is formatted ONLY IF the log message will actually be output
- Performance impact: When debug logging disabled (default), lazy % avoids unnecessary string operations

**Pattern consistency:**
This matches the established pattern from Plan 06-02 decision: "Lazy % formatting for performance (avoid f-strings in logging calls)"
