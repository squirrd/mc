1. **Add imports:**

   ```python
   import os
   import logging
   from tqdm import tqdm
   from tenacity import (
       retry,
       stop_after_attempt,
       wait_exponential,
       retry_if_exception_type,
       before_sleep_log,
   )
   import requests
   ```

2. **Create module-level logger** (if not added in Plan 2):
   ```python

   ```

3. **Replace existing download_file() method** with enhanced version following RESEARCH.md Pattern 3, 4, and 5:

   **Retry decorator** (above download_file method):
   ```python
   @retry(
       stop=stop_after_attempt(3),
       wait=wait_exponential(multiplier=1, min=1, max=4),
       retry=(
           retry_if_exception_type(requests.exceptions.Timeout) |
           retry_if_exception_type(requests.exceptions.ConnectionError)
       ),
       before_sleep=before_sleep_log(logger, logging.WARNING),
       reraise=True,
   )
   ```

   **download_file() implementation:**
   - **Check for partial file:** If local_path exists, get file size with os.path.getsize()
   - **Add Range header if resuming:** headers['Range'] = f'bytes={downloaded}-'
   - **Open file in appropriate mode:** 'ab' (append) if resuming, 'wb' (write) if new
   - **Make request with stream=True and timeout=30** (existing timeout from Phase 4)
   - **Handle 200 and 206 status codes:** 206 = Partial Content (resume), 200 = Full content
   - **Parse Content-Range header for total size** when status is 206: "bytes 1000-2000/3000" -> total is 3000
   - **Use Content-Length for total size** when status is 200
   - **Create tqdm progress bar:**
     ```python
     with open(local_path, mode) as f, tqdm(
         desc=os.path.basename(local_path),
         initial=downloaded,  # Starting position if resuming
         total=total_size,
         unit='B',
         unit_scale=True,     # Auto-scale to KB, MB, GB
         unit_divisor=1024,   # Use 1024 for binary units
         mininterval=0.1,     # Update every 100ms
     ) as pbar:
         for chunk in response.iter_content(chunk_size=8192):
             size = f.write(chunk)
             pbar.update(size)
     ```
   - **Don't retry 401/403 errors** - these are auth failures (fail fast with response.raise_for_status())
   - **Do retry 429 and 503 errors** - rate limit and service unavailable (let retry decorator handle)
   - **Raise RuntimeError on failure** (existing pattern from Phase 4)

4. **Handle 429 rate limiting** separately from retry decorator:
   - Check response.status_code == 429
   - Retry-After header may be present: `retry_after = int(response.headers.get('Retry-After', 2))`
   - Raise HTTPError to trigger retry (decorator handles exponential backoff)

5. **Update docstring** to document progress bar, retry, and resume behavior:
   ```python
   """
   Download file with progress bar, automatic retry, and resume support.

   Shows progress bar with transfer speed and ETA. Automatically retries
   on network errors (timeout, connection) and transient API failures
   (503, 429) with exponential backoff. Resumes interrupted downloads
   from last byte position using HTTP Range requests.

   Args:
       case_number: Case number for API endpoint
       attachment_id: Attachment ID
       local_path: Where to save file

   Raises:
       requests.HTTPError: On authentication failure (401, 403)
       RuntimeError: On download failure after retries
   """
   ```

Follow RESEARCH.md patterns exactly, especially "Combining Retry and Progress" example.
