<task type="auto">
  <name>Create logging infrastructure module</name>
  <files>src/mc/utils/logging.py</files>
  <action>
Create src/mc/utils/logging.py with three components following RESEARCH.md Pattern 1 and Pattern 2:

1. **JSONFormatter class** - Custom logging.Formatter that outputs JSON with:
   - timestamp (ISO 8601 format using datetime.fromtimestamp with timezone.utc)
   - level (INFO/WARNING/ERROR/DEBUG)
   - module (logger name)
   - message
   - exception (if exc_info present, use formatException())

2. **SensitiveDataFilter class** - Custom logging.Filter that redacts sensitive data:
   - Regex patterns for: password fields, Bearer tokens, api_key fields
   - redact_token() static method: show last 4 chars if >20 chars (e.g., "...7a3f"), otherwise fully redact
   - filter() method: apply patterns to record.getMessage(), update record.msg and clear record.args

3. **setup_logging() function** - Configure logging based on CLI arguments:
   - Parameters: json_logs (bool), debug (bool), debug_file (str, optional)
   - Get/create 'mc' logger (package name as root)
   - Set level to DEBUG if debug=True, otherwise INFO
   - Clear existing handlers
   - Create StreamHandler to sys.stdout
   - Apply JSONFormatter if json_logs=True, otherwise human-readable Formatter with format:
     '%(asctime)s [%(levelname)s] %(name)s: %(message)s', datefmt='%Y-%m-%d %H:%M:%S'
   - Add SensitiveDataFilter to handler
   - If debug=True and debug_file provided, create FileHandler with detailed format including line numbers
   - Set logger.propagate = False to prevent propagation to root logger
   - Return configured logger

Use stdlib logging module only (no structlog). Follow patterns from RESEARCH.md exactly.

**Module docstring:** "Logging configuration and utilities for the MC CLI application."
