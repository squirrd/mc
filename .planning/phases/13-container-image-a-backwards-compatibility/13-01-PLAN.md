Replace existing container/Containerfile with RHEL 10 UBI-based image following research patterns:

**Base image:** `FROM registry.access.redhat.com/ubi10/ubi:10.1`

**Essential tools installation (single RUN layer with cleanup):**
- Python: python3.11 python3-pip (match host version from pyproject.toml)
- Text editors: vim nano
- Networking: curl wget
- Security: openssl (pre-installed in UBI)
- Data serialization: jq (if available in UBI repos, otherwise document for runtime install)
- File operations: tar gzip (pre-installed in UBI)

**Package manager setup:**
- Include dnf/yum for runtime package installation
- Clean dnf cache in same RUN command: `&& dnf clean all`

**MC CLI installation:**
- Create /opt/mc directory for CLI installation
- Copy project files: `COPY . /opt/mc`
- Install in editable mode: `RUN pip3 install --no-cache-dir --no-build-isolation -e /opt/mc`
- Why editable install: Allows code changes without image rebuild (development-oriented container)
- Why --no-build-isolation: Prevents runtime import errors per research pitfall

**Container user setup:**
- Create non-root user: `RUN groupadd --system mcuser && useradd --system --gid mcuser --home-dir /home/mcuser --create-home mcuser`
- Create /case mount point with correct ownership: `RUN mkdir -p /case && chown mcuser:mcuser /case`
- Switch to non-root: `USER mcuser`
- Why non-root: Security best practice, works with --userns=keep-id from Phase 11

**Environment setup:**
- Set MC_RUNTIME_MODE=agent environment variable
- Set WORKDIR /case (start in workspace)

**Entrypoint:**
- Copy entrypoint.sh to /usr/local/bin/entrypoint.sh
- Use exec form for proper signal handling: `ENTRYPOINT ["/usr/local/bin/entrypoint.sh"]`
- Default command: `CMD ["/bin/bash"]`

**Layer optimization:**
- Combine related RUN commands to minimize layers
- Clean package manager cache in same RUN as install
- Follow research Pattern 4 (Layer Optimization)
  </action>
