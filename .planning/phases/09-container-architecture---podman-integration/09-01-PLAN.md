<task type="auto">
  <name>Task 2: Create platform detection module</name>
  <files>src/mc/integrations/platform_detect.py</files>
  <action>
Create src/mc/integrations/platform_detect.py implementing platform detection and Podman availability checking.

Required functions (follow patterns from RESEARCH.md):

1. **detect_platform() -> str**
   - Use platform.system() to detect 'Darwin' (macOS) vs 'Linux'
   - Return 'macos', 'linux', or 'unsupported'
   - Simple, no caching

2. **is_podman_machine_running() -> bool**
   - macOS only: run `podman machine list --format json`
   - Parse JSON, check if any machine has "Running": true
   - Return False on any error (FileNotFoundError, CalledProcessError, JSON parse error)

3. **ensure_podman_ready(platform_type: str) -> None**
   - For macOS: check machine status, prompt user if stopped ("Podman machine is stopped. Start it? [y/n]")
   - If yes: run `podman machine start`, if no: raise RuntimeError
   - For Linux: verify `podman --version` succeeds
   - For unsupported: raise RuntimeError with helpful message pointing to docs
   - Use subprocess.run with check=True for operations that must succeed

4. **get_socket_path(platform_type: str) -> Optional[str]**
   - Check CONTAINER_HOST environment variable first (return path without unix:// prefix)
   - For Linux rootless: /run/user/{uid}/podman/podman.sock (use os.getuid())
   - Check XDG_RUNTIME_DIR as alternative
   - Fallback to rootful socket /run/podman/podman.sock if rootless doesn't exist
   - For macOS: return None (podman-py auto-detects)
   - Return str path or None

5. **check_podman_version(warn_threshold: int = 3, fail_threshold: int = 7) -> dict**
   - Run `podman --version`, parse version with regex
   - Compare to expected version (5.0) using sliding window logic from RESEARCH.md
   - Return dict with keys: version, major, minor, action ('ok'/'warn'/'fail'), message
   - Handle FileNotFoundError (Podman not installed) as action='fail'

Follow existing codebase patterns:
- Module docstring: """Platform detection and Podman availability checking."""
- Function docstrings with Args, Returns, Raises sections (Google style)
- Use subprocess.run with capture_output=True, text=True
- Error handling with try/except for subprocess operations
- Use pathlib.Path for socket path checking
- Import json for parsing Podman machine list output
- Use type hints (platform_type: str, etc.) per v1.0 type safety work

Apply research patterns from 09-RESEARCH.md sections: Pattern 2 (platform detection), Pattern 3 (socket path detection), Pattern 5 (version checking).
