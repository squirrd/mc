  <action>
Enhance build-container.sh with registry query capability using skopeo. Follow the patterns from 23-RESEARCH.md.

**Add these functions (after version extraction, before build orchestration section):**

1. `query_latest_registry_version()` - Query registry for highest semantic version tag
   - Use skopeo list-tags with REGISTRY_REPO variable (e.g., "quay.io/dsquirre/mc-rhel10")
   - Parse JSON with jq, filter to semver format (grep -E '^[0-9]+\.[0-9]+\.[0-9]+$')
   - Sort with `sort -V`, take highest with `tail -1`
   - Return empty string if no versions published (not an error)
   - Implement exponential backoff with retry-after header support (5 max attempts)
   - Fail on authentication errors, network errors, malformed JSON

2. `check_version_exists()` - Check if specific version exists on registry
   - Use skopeo inspect docker://REGISTRY_REPO:VERSION
   - Return 0 if exists, 1 if not found
   - Silent operation (stderr to /dev/null)

3. `get_registry_digest()` - Get manifest digest for specific version
   - Use skopeo inspect, parse .Digest field with jq -r
   - Return digest or empty string if version not found
   - Follow RESEARCH.md warning: Use manifest-level digest, not RepoDigests

4. `compare_with_registry()` - Build image and compare with registry digest
   - Build image first (using current versions.yaml) to get local digest
   - Get local image digest with `podman inspect IMAGE_TAG | jq -r '.[0].Digest'`
   - Query registry digest with get_registry_digest
   - Compare digests (exact string match)
   - Output comparison results:
     - "Latest on quay.io: VERSION" (or "No versions published")
     - "Local version: IMAGE_VERSION"
     - "Digest comparison: MATCH|DIFFER" with digests shown in verbose mode
   - Return 0 if match, 1 if differ

**Add global variables (top of file):**
```bash
REGISTRY_REPO="${REGISTRY_REPO:-quay.io/dsquirre/mc-rhel10}"
JSON_OUTPUT=false
```

**Update preflight_checks() function:**
- Add check for skopeo: `command -v skopeo`
- Add check for jq: `command -v jq`
- Both fail with actionable install instructions if missing

**Update argument parsing (while loop):**
- Add --json flag that sets JSON_OUTPUT=true

**Error handling principles (from CONTEXT.md):**
- Network failures: Fail the build (hard error)
- Rate limiting: Retry with exponential backoff, then fail
- Authentication failures: Fail the build (invalid credentials are errors)
- Unexpected errors (500, invalid JSON): Fail the build (no retries for server errors)
- Use `set -euo pipefail` to fail on any command error

**Credential reuse (from RESEARCH.md):**
- Check for auth.json in both locations:
  - macOS/persistent: $HOME/.config/containers/auth.json
  - Linux ephemeral: ${XDG_RUNTIME_DIR}/containers/auth.json
- skopeo automatically uses credentials if auth.json exists
- Try anonymous access for public repos if no credentials
- No explicit auth flags needed (skopeo handles this)

**Important: Do NOT implement version bumping in this task.**
Version bumping is Phase 24. This task only queries and compares.

**Update main execution section (after architecture detection, before build):**

Add registry query step:
```bash
# Query registry for version staleness
if [[ "$DRY_RUN" != "true" ]]; then
  echo "Querying quay.io registry..."

  # Get latest registry version
  LATEST_REGISTRY_VERSION=$(query_latest_registry_version)

  # Check if current version exists on registry
  if check_version_exists "$IMAGE_VERSION"; then
    VERSION_EXISTS=true
    REGISTRY_DIGEST=$(get_registry_digest "$IMAGE_VERSION")
  else
    VERSION_EXISTS=false
    REGISTRY_DIGEST=""
  fi

  # Display comparison results
  if [[ -n "$LATEST_REGISTRY_VERSION" ]]; then
    echo "  Latest on quay.io: $LATEST_REGISTRY_VERSION"
  else
    echo "  Latest on quay.io: No versions published"
  fi

  echo "  Local version: $IMAGE_VERSION"

  if [[ "$VERSION_EXISTS" == "true" ]]; then
    echo "  Version $IMAGE_VERSION exists on registry"
    if [[ "$VERBOSE" == "true" ]]; then
      echo "  Registry digest: $REGISTRY_DIGEST"
    fi
  else
    echo "  Version $IMAGE_VERSION not found on registry (new version)"
  fi
fi
```

$(for tool_name in "${!TOOL_VERSIONS[@]}"; do

}
EOF
}
