## Decisions Made

**ConfigManager.get() implementation:**
- Uses dotted key paths (e.g., 'workspace.base_directory') for nested config access
- Returns default value on FileNotFoundError, KeyError, or TypeError
- Allows container commands to work even when config file missing or incomplete

**Authentication separation:**
- Container lifecycle commands (create, list, stop, delete, exec) never initialize SalesforceAPIClient
- Only terminal attachment (case_terminal, quick_access) requires Salesforce credentials
- Salesforce credentials validated only at terminal attachment time (lazy validation)

**Documentation location:**
- Authentication architecture documented in container.py module docstring (not inline comments)
- Module docstring survives linter/formatter changes better than inline comments
