<task type="auto">
  <name>Task 2: Implement macOS and Linux terminal launchers</name>
  <files>
    src/mc/terminal/macos.py
    src/mc/terminal/linux.py
  </files>
  <action>
Implement platform-specific terminal launchers with non-blocking execution and proper escaping.

**src/mc/terminal/macos.py:**
- MacOSLauncher class implementing TerminalLauncher interface
- Support iTerm2 and Terminal.app
- Use AppleScript via osascript for terminal launching (subprocess.Popen for non-blocking)
- AppleScript pattern: activate application, create window, write text (RESEARCH.md examples)
- Escape AppleScript strings: escape backslashes and quotes (RESEARCH.md Pitfall 6)
- iTerm2 script: `tell application "iTerm2" activate; create window; tell current session write text "{command}"`
- Terminal.app script: `tell application "Terminal" activate; do script "{command}"`
- Set window title via tab name (iTerm2) or custom title (Terminal.app)

**src/mc/terminal/linux.py:**
- LinuxLauncher class implementing TerminalLauncher interface
- Support gnome-terminal, konsole, xfce4-terminal
- Use direct subprocess.Popen with terminal-specific args
- gnome-terminal: `gnome-terminal --title="{title}" -- bash -c "{command}"`
- konsole: `konsole --noclose -e bash -c "{command}"` (title via escape sequences)
- xfce4-terminal: `xfce4-terminal --title="{title}" --command="bash -c '{command}'"`
- Non-blocking: Use Popen with background cleanup (goroutine pattern from RESEARCH.md)

Handle missing terminal binaries gracefully: check shutil.which() before launching, raise clear error if not found.
Follow RESEARCH.md code examples for each terminal type.
  </action>
  <verify>
python3 -c "from mc.terminal.macos import MacOSLauncher; from mc.terminal.linux import LinuxLauncher; print('Platform launchers imported')"
  </verify>
  <done>
macOS launcher supports iTerm2 and Terminal.app with AppleScript automation, Linux launcher supports gnome-terminal/konsole/xfce4-terminal with direct CLI invocation, both use non-blocking subprocess execution.
  </done>
