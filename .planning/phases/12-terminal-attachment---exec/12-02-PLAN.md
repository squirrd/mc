Create shell customization module for generating custom bashrc to inject into containers via BASH_ENV.

**src/mc/terminal/shell.py:**

generate_bashrc(case_number: str, case_metadata: dict[str, Any]) -> str:
- Generate custom bashrc content as string
- Set PS1 prompt: `export PS1='[MC-{case_number}] \w\$ '`
- Define helper aliases: `alias ll='ls -lah'`, `alias case-info='echo "Case: ${MC_CASE_ID}"'`
- Define helper function mc-help() showing available commands
- Export case metadata as environment variables: MC_CASE_ID, MC_CUSTOMER, MC_DESCRIPTION
- Call generate_banner() from banner module to display welcome banner on shell init
- Follow RESEARCH.md custom bashrc example

get_bashrc_path(case_number: str) -> str:
- Return path where bashrc should be written: `~/.mc/bashrc/mc-{case_number}.bashrc`
- Use platformdirs or os.path.expanduser for cross-platform home directory handling
- Create parent directory if missing (os.makedirs with exist_ok=True)

write_bashrc(case_number: str, case_metadata: dict[str, Any]) -> str:
- Generate bashrc content using generate_bashrc()
- Get path using get_bashrc_path()
- Write content to file
- Return absolute path (for use with BASH_ENV environment variable)
- Set file permissions to 0o644 (readable by all, writable by owner)

Follow established MC patterns: use platformdirs for config locations (from Phase 11), use type hints, docstrings.
  </action>

  </verify>
  <done>

  </done>

**src/mc/terminal/banner.py:**

generate_banner(case_metadata: dict[str, Any]) -> str:
- Format case metadata into readable welcome banner
- Banner structure (per CONTEXT.md decisions):
  ```
  =================================================
  Case: {case_number}
  Customer: {customer_name}
  Description: {case_description}

  Summary:
  {case_summary}

  Next Steps:
  {case_next_steps}
  =================================================
  ```
- Handle missing fields gracefully: use "N/A" or skip section if metadata not available
- Wrap long descriptions to 80 characters for readability (textwrap module)
- DO NOT include comments (CONTEXT.md: "Comments stored as metadata but NOT displayed in banner")
- Return formatted string with newlines

format_field(value: str | None, max_width: int = 80) -> str:
- Helper function to wrap text to specified width
- Handle None values (return "N/A")
- Use textwrap.fill() for wrapping

Follow MC conventions: use type hints, docstrings, simple string formatting with f-strings.
Case metadata structure from Phase 10 (SalesforceAPIClient): case_number, customer_name, description, summary, next_steps.
