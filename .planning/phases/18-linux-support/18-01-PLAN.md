<task type="auto">
  <name>Task 2: Add window ID capture, validation, and focusing methods</name>
  <files>src/mc/terminal/linux.py</files>
  <action>
Add window management methods to LinuxLauncher (after launch() method):

1. Add _capture_window_id() method:
   - Add time.sleep(0.5) delay for window creation (matches Phase 16 macOS pattern)
   - Check shutil.which("xdotool") - return None if not available
   - Run subprocess.run(["xdotool", "getactivewindow"], capture_output=True, text=True, timeout=5)
   - Parse result.stdout.strip() as int (decimal window ID)
   - Convert decimal to hex format: f"0x{window_id:08x}" for consistency with wmctrl output
   - Return hex string or None on failure
   - Wrap in try/except, return None on any exception
   - Use RESEARCH.md Pattern 4 (lines 210-244)

2. Add _window_exists_by_id(window_id: str) method:
   - Check shutil.which("wmctrl") - return False if not available
   - Run subprocess.run(["wmctrl", "-l"], capture_output=True, text=True, timeout=5)
   - Check if window_id.lower() in result.stdout.lower() (wmctrl -l lists all windows, first column is window ID in hex)
   - Return True if found, False otherwise
   - Wrap in try/except, return False on any exception
   - Use RESEARCH.md Pattern 5 (lines 250-277)

3. Add focus_window_by_id(window_id: str) method:
   - Check shutil.which("wmctrl") - return False if not available
   - Run subprocess.run(["wmctrl", "-i", "-a", window_id], capture_output=True, text=True, timeout=5)
   - The -i flag interprets window argument as numeric ID (not name)
   - The -a flag activates window: switches to desktop/workspace, raises window, and focuses
   - Return True if returncode == 0, False otherwise
   - Catch subprocess.TimeoutExpired explicitly, return False
   - Wrap in try/except, return False on any exception
   - Use RESEARCH.md Pattern 6 (lines 285-313)

4. Match MacOSLauncher API surface for cross-platform compatibility:
   - All three methods should have docstrings matching Phase 16 macos.py style
   - _capture_window_id() -> str | None (matches MacOSLauncher)
   - _window_exists_by_id(window_id: str) -> bool (matches MacOSLauncher)
   - focus_window_by_id(window_id: str) -> bool (matches MacOSLauncher)

Why these implementations: RESEARCH.md identifies wmctrl as more reliable than xdotool for ID-based focusing (line 37). The -i -a flag combination handles workspace switching automatically (EWMH/NetWM standard). xdotool getactivewindow for capture is necessary because gnome-terminal doesn't set $WINDOWID (RESEARCH.md Pitfall 3). 0.5s delay matches Phase 16's proven pattern for reliable window ID capture.
