**create() method:**
```python
def create(self, case_number: str, workspace_path: str, customer_name: str = "Unknown") -> Container:
    """Create container for case with workspace mounted at /case.

    Args:
        case_number: Case number (e.g., "12345678")
        workspace_path: Host path to mount as /case (e.g., "/Users/user/Cases/12345678")
        customer_name: Customer name for container label (optional, defaults to "Unknown")

    Returns:
        podman.Container instance (running container)

    Raises:
        RuntimeError: If container creation fails
    """
```

**Implementation (following Pattern 1 from RESEARCH.md):**
1. Call self._reconcile() to detect external deletions
2. Check if container exists in state via self.state.get_container(case_number)
3. If exists, check Podman status via self.podman.client.containers.get(container_id)
4. If stopped/exited, restart with container.start() and print "Restarting container for case {case_number}..."
5. If running, return existing container
6. If not in state, create new container:
   - Image: "registry.access.redhat.com/ubi9/ubi:latest"
   - Name: f"mc-{case_number}"
   - Labels: {"mc.managed": "true", "mc.case_number": case_number, "mc.customer": customer_name}
   - Volumes: {workspace_path: {"bind": "/case", "mode": "rw"}}
   - userns_mode: "keep-id" (CRITICAL for rootless permissions - see RESEARCH.md Pitfall 1)
   - tty: True, stdin_open: True (for interactive shell)
   - detach: True
   - command: ["/bin/bash", "-l"] (login shell to load profile)
7. Call container.start()
8. Record in state: self.state.add_container(case_number, container.id, workspace_path)
9. Return container

**_reconcile() helper:**
```python
def _reconcile(self):
    """Synchronize state with Podman to detect external deletions."""
    # List all mc-managed containers from Podman
    podman_containers = self.podman.client.containers.list(
        all=True,  # Include stopped
        filters={"label": "mc.managed=true"}
    )
    podman_ids = {c.id for c in podman_containers}

    # Reconcile state database
    self.state.reconcile(podman_ids)
```

Handle exceptions:
- Catch podman-py exceptions (APIError, ContainerError) and wrap in RuntimeError with helpful message
- Catch state database errors and wrap in RuntimeError
- Use platform-specific error messages from PodmanClient if connection fails

```python
import pytest
from mc.integrations.podman import PodmanClient
from mc.container.state import StateDatabase
from mc.container.manager import ContainerManager
import tempfile
import os

@pytest.mark.integration
@pytest.mark.skipif(not _podman_available(), "Podman not available")
def test_create_container_e2e():
    """End-to-end test: create container, verify it exists, cleanup."""
    # Setup
    with tempfile.TemporaryDirectory() as tmpdir:
        db_path = os.path.join(tmpdir, "test.db")
        workspace_path = os.path.join(tmpdir, "workspace")
        os.makedirs(workspace_path)

        client = PodmanClient()
        state_db = StateDatabase(db_path)
        manager = ContainerManager(client, state_db)

        # Create container
        container = manager.create("99999999", workspace_path, "TestCustomer")

        # Verify container exists and is running
        assert container.status == "running"
        assert "mc.managed" in container.labels
        assert container.labels["mc.case_number"] == "99999999"

        # Verify state database updated
        metadata = state_db.get_container("99999999")
        assert metadata is not None
        assert metadata.container_id == container.id

        # Cleanup
        container.stop(timeout=2)
        container.remove()
        state_db.delete_container("99999999")
```
