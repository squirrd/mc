  <action>
Create StateDatabase class in src/mc/container/state.py implementing:

**Initialization:**
- __init__(db_path: str) - store path, call _setup_wal_mode()
- _setup_wal_mode() - enable WAL mode, set PRAGMA synchronous=NORMAL, cache_size=10000
- _ensure_schema() - create containers table with schema from RESEARCH.md (case_number PRIMARY KEY, container_id NOT NULL, workspace_path NOT NULL, created_at INTEGER, updated_at INTEGER), create indices on container_id and created_at

**Database location:**
Use platformdirs (already in dependencies from Phase 3) to store database in platform-appropriate location:
```python
from platformdirs import user_data_dir
data_dir = user_data_dir("mc", "redhat")
db_path = os.path.join(data_dir, "containers.db")
```

**Context manager pattern:**
- _connection() - context manager yielding sqlite3.Connection with row_factory=sqlite3.Row, auto-commit on success, rollback on exception, always close

**CRUD operations:**
- add_container(case_number, container_id, workspace_path) - INSERT with current timestamp for created_at/updated_at
- get_container(case_number) -> ContainerMetadata | None - SELECT by case_number, return dataclass or None
- list_all() -> list[ContainerMetadata] - SELECT all ordered by created_at DESC
- delete_container(case_number) - DELETE by case_number
- update_container(case_number, **kwargs) - UPDATE updated_at and any provided fields

**Reconciliation:**
- reconcile(podman_container_ids: set[str]) - Accept set of container IDs from Podman, delete any state entries with container_id NOT in provided set (orphaned state cleanup)

Use sqlite3.connect with timeout=30.0 to handle concurrent access gracefully.

Follow RESEARCH.md Pattern 3 for WAL setup, Pattern 2 for reconciliation.
