Create WindowRegistry class following the exact StateDatabase pattern from src/mc/container/state.py with these specifications:

**Class structure:**
- `__init__(db_path: str | None = None)` - accepts optional path, defaults to platformdirs.user_data_dir("mc", "redhat") + "window.db"
- `_memory_conn` attribute for persistent :memory: database connections (testing support)
- `_setup_wal_mode()` method - configures PRAGMA journal_mode=WAL, synchronous=NORMAL, busy_timeout=30000, cache_size=-64000
- `_ensure_schema()` method - creates window_registry table with PRIMARY KEY on case_number
- `_connection()` context manager - auto-commit on success, rollback on exception, handles both :memory: and file databases

**Database schema:**
```sql
CREATE TABLE IF NOT EXISTS window_registry (
    case_number TEXT PRIMARY KEY,
    window_id TEXT NOT NULL,
    created_at INTEGER NOT NULL,
    terminal_type TEXT,
    last_validated INTEGER
)
CREATE INDEX IF NOT EXISTS idx_window_id ON window_registry(window_id)
```

**CRUD operations:**

1. `register(case_number: str, window_id: str, terminal_type: str) -> bool`
   - INSERT INTO window_registry with current timestamp for both created_at and last_validated
   - Catch sqlite3.IntegrityError (UNIQUE constraint violation) and return False
   - Return True on successful registration (first-write-wins behavior)

2. `lookup(case_number: str, validator: Callable[[str], bool]) -> str | None`
   - SELECT window_id WHERE case_number = ?
   - Call validator(window_id) to check if window still exists
   - If validator returns False: DELETE the entry and return None (auto-cleanup)
   - If validator returns True: UPDATE last_validated timestamp and return window_id
   - Return None if case_number not found

3. `remove(case_number: str) -> None`
   - DELETE FROM window_registry WHERE case_number = ?
   - No return value (idempotent operation)

**Why these choices:**
- WAL mode: Enables concurrent reads with single writer, prevents "database locked" errors
- synchronous=NORMAL: Best balance for WAL mode (survives app crashes, adequate durability)
- busy_timeout=30000: 30-second timeout for lock acquisition (matches StateDatabase)
- TEXT window_id: Supports both numeric (macOS) and alphanumeric (Linux) identifiers
- Lazy validation: Only validate on lookup, not on every read (performance)
- Auto-cleanup: Self-healing registry removes stale entries transparently
- First-write-wins: UNIQUE constraint handles race conditions without application-level locking
