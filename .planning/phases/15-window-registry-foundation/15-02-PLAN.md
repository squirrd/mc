Add two new methods to the MacOSLauncher class for window ID operations:

**1. _capture_window_id() -> str | None**

Add method to capture window ID after terminal creation (called after AppleScript creates window).

For iTerm2:
```applescript
tell application "iTerm"
    -- After creating window
    return id of current window as text
end tell
```

For Terminal.app:
```applescript
tell application "Terminal"
    -- After creating window
    return id of front window as text
end tell
```

Implementation:
- Accept no parameters (captures "current window" or "front window" depending on terminal)
- Return window ID as string, or None if capture fails
- Use existing `_escape_applescript` method if needed
- Timeout after 5 seconds (match existing patterns)
- Handle exceptions gracefully (return None)

**2. _window_exists_by_id(window_id: str) -> bool**

Add method to validate if window with specific ID still exists.

For iTerm2:
```applescript
tell application "iTerm"
    try
        repeat with theWindow in windows
            if (id of theWindow as text) is "TARGET_ID" then
                return true
            end if
        end repeat
        return false
    on error
        return false
    end try
end tell
```

For Terminal.app:
```applescript
tell application "Terminal"
    try
        repeat with theWindow in windows
            if (id of theWindow as text) is "TARGET_ID" then
                return true
            end if
        end repeat
        return false
    on error
        return false
    end try
end tell
```

Implementation:
- Accept window_id as string parameter
- Return True if window exists, False otherwise
- Escape window_id using `_escape_applescript`
- Check if osascript available (return False if missing)
- Timeout after 5 seconds
- Handle exceptions gracefully (return False)

**Why these methods:**
- _capture_window_id: Phase 16 will use this to store window IDs in registry after creation
- _window_exists_by_id: Used as validator callback for WindowRegistry.lookup() operations
- Separate from launch(): Clean separation of concerns, launch() stays focused on window creation

**Placement:** Add these methods after `focus_window_by_title()` and before `_build_iterm_script()` in the class.

Do NOT modify the launch() method yet - that integration happens in Phase 16. These are standalone utility methods.
