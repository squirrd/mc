Create `tests/unit/test_terminal_macos_api.py` with tests covering the new Python API path in MacOSLauncher. Mock the iterm2 module at the module level — no real iTerm2 needed.

**Mocking strategy:** Patch `mc.terminal.macos._ITERM2_LIB_AVAILABLE` and `mc.terminal.macos._iterm2_module` (or `iterm2` within the function scope). For async API calls use `unittest.mock.AsyncMock`.

Key test scenarios to cover (write one test function per scenario):

**_try_iterm2_api — API available and succeeds:**
- Mock `_ITERM2_LIB_AVAILABLE = True`, mock `_launch_via_iterm2_api` returning `"w1234-uuid"`
- Assert `_try_iterm2_api(options)` returns `"w1234-uuid"`

**_try_iterm2_api — library not available:**
- Mock `_ITERM2_LIB_AVAILABLE = False`
- Assert `_try_iterm2_api(options)` returns `None` without calling any iterm2 code

**_try_iterm2_api — API raises exception (connection refused):**
- Mock `_ITERM2_LIB_AVAILABLE = True`, mock `_launch_via_iterm2_api` raising `OSError("connection refused")`
- Assert `_try_iterm2_api(options)` returns `None` (does not re-raise)

**launch() — API succeeds, returns without Popen:**
- Mock `_try_iterm2_api` returning `"w1234-uuid"` on a `MacOSLauncher(terminal="iTerm2")` instance
- Patch `subprocess.Popen`
- Call `launch(options)`
- Assert `subprocess.Popen` was NOT called (API path used instead)
- Assert `launcher._last_api_window_id == "w1234-uuid"`

**launch() — API fails, falls back to Terminal.app AppleScript:**
- Mock `_try_iterm2_api` returning `None` on a `MacOSLauncher(terminal="iTerm2")` instance
- Patch `subprocess.Popen` and `shutil.which` returning `/usr/bin/osascript`
- Patch `_should_show_iterm2_fallback_notice` returning `False` (suppress stderr)
- Call `launch(options)`
- Assert `subprocess.Popen` WAS called with `["osascript", "-e", ...]`
- Assert the script contains `"Terminal"` (fell back to Terminal.app AppleScript)

**launch() — API fails, shows fallback notice on first occurrence:**
- Mock `_try_iterm2_api` returning `None`
- Patch `_should_show_iterm2_fallback_notice` returning `True`
- Patch `_record_iterm2_fallback_notice`
- Patch `subprocess.Popen`, `shutil.which` returning path
- Capture stderr
- Assert stderr contains "iTerm2 API unavailable, using Terminal.app"
- Assert stderr contains "Enable via iTerm2 > Settings > General > Magic"
- Assert `_record_iterm2_fallback_notice` was called once

**launch() — API fails, fallback notice suppressed same day:**
- Mock `_try_iterm2_api` returning `None`
- Patch `_should_show_iterm2_fallback_notice` returning `False`
- Patch `subprocess.Popen`, `shutil.which` returning path
- Capture stderr
- Assert stderr does NOT contain "iTerm2 API unavailable"

**_capture_window_id() — returns API window_id when set:**
- Create `MacOSLauncher(terminal="iTerm2")`
- Set `launcher._last_api_window_id = "w1234-uuid"`
- Call `_capture_window_id()`
- Assert returns `"w1234-uuid"`
- Assert `launcher._last_api_window_id` is `None` after call (consumed)

**_window_exists_by_id() — uses API for iTerm2 when available:**
- Mock `_ITERM2_LIB_AVAILABLE = True`
- Mock `_window_exists_by_id_api` returning `True`
- Call `_window_exists_by_id("w1234-uuid")` on `MacOSLauncher(terminal="iTerm2")`
- Assert returns `True` without calling `osascript`

**_window_exists_by_id() — falls back to AppleScript when API returns None:**
- Mock `_window_exists_by_id_api` returning `None` (API unavailable)
- Patch `subprocess.run` returning osascript "true" result
- Call `_window_exists_by_id("12345")` on `MacOSLauncher(terminal="iTerm2")`
- Assert `subprocess.run` was called (fell back to AppleScript)

**focus_window_by_id() — uses API for iTerm2 when available:**
- Mock `_focus_window_by_id_api` returning `True`
- Call `focus_window_by_id("w1234-uuid")` on `MacOSLauncher(terminal="iTerm2")`
- Assert returns `True` without calling `osascript`

**_should_show_iterm2_fallback_notice() and _record_iterm2_fallback_notice():**
- Test that `_should_show_iterm2_fallback_notice` returns `True` when sentinel file missing
- Test that after `_record_iterm2_fallback_notice`, `_should_show_iterm2_fallback_notice` returns `False`
  (use tmp_path fixture to redirect `_ITERM2_FALLBACK_SENTINEL` for isolation)

Use `from unittest.mock import AsyncMock, MagicMock, Mock, patch` throughout.
Use `mocker` fixture (pytest-mock) where it simplifies patching instance methods.
Name tests: `test_<method>_<scenario>_<expected>`.
