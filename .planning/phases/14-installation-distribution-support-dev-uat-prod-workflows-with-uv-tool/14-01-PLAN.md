<task type="auto">
  <name>Task 2: Document installation workflows for dev/UAT/prod</name>
  <files>INSTALL.md, README.md</files>
  <action>
Create INSTALL.md with three workflow sections matching research patterns:

1. **Prerequisites** section:
   - Python 3.11+ requirement
   - uv installation (curl command, Homebrew alternative)
   - Shell PATH configuration (`uv tool update-shell`)

2. **Development Workflow** section:
   - Use `uv run` for all development tasks (no manual venv activation)
   - Example commands: `uv run mc --help`, `uv run pytest`, `uv run mypy src/`
   - Adding dependencies: `uv add <package>`
   - Upgrading dependencies: `uv lock --upgrade-package <package>`
   - Note: First `uv run` auto-creates .venv and syncs from uv.lock

3. **UAT Workflow** section:
   - Install from local directory: `uv tool install -e .` (editable)
   - Test installation: `mc --version`, `mc --help`
   - View installed tools: `uv tool list --show-paths`
   - Uninstall when done: `uv tool uninstall mc-cli`
   - Note: Changes to code reflected immediately (editable mode)

4. **Production Workflow** section:
   - Install from git: `uv tool install git+https://github.com/user/mc-cli.git`
   - Install specific version: `uv tool install git+https://github.com/user/mc-cli.git@v2.0.0`
   - Upgrade: `uv tool upgrade mc-cli`
   - Uninstall: `uv tool uninstall mc-cli`
   - Note: When published to PyPI, use `uv tool install mc-cli`

5. **Troubleshooting** section:
   - PATH not configured: Run `uv tool update-shell` or manually add ~/.local/bin
   - Conda interference: Use `uv run`/`uv sync` instead of `uv pip`
   - Changes not reflected: After modifying pyproject.toml, run `uv sync`

Update README.md:
- Add "Installation" section near the top with quick start
- Link to INSTALL.md for detailed instructions
- Show simplest install path: `uv tool install git+https://github.com/user/mc-cli.git` (or PyPI when published)
- One-line verify: `mc --version`

Do NOT include setup.py references (deprecated). Do NOT mention pip install -e . (use uv equivalents).
  </action>
