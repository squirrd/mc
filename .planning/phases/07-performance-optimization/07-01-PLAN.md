  <action>
Create `src/mc/utils/cache.py` following the pattern from Phase 4 token caching (04-01-SUMMARY.md):

**File-based cache structure:**
- Cache directory: `~/.mc/cache/` with 0o700 permissions
- Cache files: `case_{case_number}.json` containing `{data: {case_details, account_details}, cached_at: timestamp}`
- TTL: 30 minutes (1800 seconds)

**Core functions:**

1. `get_case_metadata(case_number, api_client, force_refresh=False)`
   - Check cache file exists and is within TTL
   - Return cached data if valid, otherwise fetch from API
   - Return tuple: `(case_details, account_details, was_cached: bool)`
   - Use `api_client.fetch_case_details()` and `api_client.fetch_account_details()`

2. `cache_case_metadata(case_number, case_details, account_details)`
   - Create cache directory with secure permissions (mkdir with mode=0o700)
   - Write JSON with structure: `{data: {case_details, account_details}, cached_at: time.time()}`
   - Use atomic write pattern (write to temp file, rename)

3. `get_cached_age_minutes(case_number)`
   - Return age in minutes if cache exists, None otherwise
   - Used for displaying "(cached Xm ago)" indicator

**Error handling:**
- Handle missing cache gracefully (fetch from API)
- Handle corrupted JSON (delete cache, fetch from API)
- Log cache hits/misses at DEBUG level

**Testing (tests/unit/test_cache.py):**
- Test cache hit within TTL window
- Test cache miss after TTL expiration
- Test fresh API call on cache miss
- Test force_refresh bypasses cache
- Test cache age calculation
- Use tmp_path fixture for cache directory isolation
- Mock time.time() for TTL testing
