  <name>Implement token caching with expiration validation</name>
  <files>src/mc/utils/auth.py</files>
  <action>
Add token caching to `get_access_token()` function following the research pattern:

1. Define constants at module level:
   - `TOKEN_CACHE_PATH = os.path.expanduser("~/.mc/token")`
   - `EXPIRY_BUFFER_SECONDS = 300` (5-minute buffer)
   - `DEFAULT_TTL_SECONDS = 3600` (1 hour default)

2. Create helper function `is_token_expired(expires_at, buffer_seconds=300)`:
   - Returns True if current time >= (expires_at - buffer_seconds)
   - Uses `time.time()` for current timestamp

3. Create helper function `load_token_cache()`:
   - Check if TOKEN_CACHE_PATH exists
   - If exists, load JSON and return dict with {access_token, expires_at, token_type}
   - If not exists or invalid JSON, return None
   - Catch and ignore JSON decode errors (corrupted cache)

4. Create helper function `save_token_cache(access_token, expires_in=None)`:
   - Calculate expires_at: `time.time() + (expires_in or DEFAULT_TTL_SECONDS)`
   - Create cache dict: {access_token, expires_at, token_type: "Bearer"}
   - Ensure ~/.mc directory exists with 0700 permissions: `os.makedirs(cache_dir, mode=0o700, exist_ok=True)`
   - Write atomically with secure permissions using pattern from RESEARCH.md:
     ```python
     fd = os.open(TOKEN_CACHE_PATH, os.O_CREAT | os.O_WRONLY | os.O_TRUNC, 0o600)
     with os.fdopen(fd, 'w') as f:
         json.dump(cache, f, indent=2)
     ```
   - Verify permissions are 600 after write

5. Update `get_access_token()` function:
   - Load cache via `load_token_cache()`
   - If cache exists and not expired, return cached token
   - If cache doesn't exist or is expired, fetch new token via SSO (existing code)
   - Extract expires_in from SSO response: `response.json().get('expires_in')`
   - Save new token to cache via `save_token_cache()`
   - Return access_token

6. Add imports: `import json`, `import time`

7. Update docstring to mention caching: "Get Red Hat API access token, using cache if valid"

Why this approach:
- File-based cache is simple, no external dependencies (no keyring library complexity)
- Atomic file write with os.open prevents race conditions and ensures secure permissions from creation
- 5-minute expiry buffer prevents using tokens that expire mid-request
- 1-hour default TTL handles cases where API doesn't provide expires_in
