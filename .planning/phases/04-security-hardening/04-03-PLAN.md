<task type="auto">
  <name>Add Bandit dependency and configuration</name>
  <files>pyproject.toml, .bandit</files>
  <action>
Configure Bandit security linter for the project:

**pyproject.toml**:

1. Add bandit to dev dependencies:
   - In `[project.optional-dependencies]` section under `dev = [...]`
   - Add line: `"bandit>=1.7.0",`
   - Maintain alphabetical order with other dev dependencies

2. Add bandit configuration section at end of file:
   ```toml
   [tool.bandit]
   exclude_dirs = ["/tests/", "/docs/", "/.planning/"]
   skips = []
   ```

**.bandit** (new file at project root):

1. Create configuration file:
   ```yaml
   # Bandit security linter configuration
   # https://bandit.readthedocs.io/

   # Exclude directories
   exclude_dirs:
     - /tests/
     - /docs/
     - /.planning/
     - /.claude/

   # Skip specific test IDs (none initially - address issues as they arise)
   skips: []

   # Only fail on HIGH severity issues
   severity: HIGH
   confidence: MEDIUM

   # Tests to run (default: all)
   tests: []
   ```

3. Install bandit:
   - Run: `pip install -e ".[dev]"` to install with dev dependencies including bandit

Why this approach:
- Bandit is industry standard for Python security scanning (used by OpenStack)
- YAML config file is more readable and maintainable than command-line flags
- HIGH severity focus prevents noise from low-priority warnings
- Empty skips list ensures we address all issues, not hide them
- Excluding test directories prevents false positives on test fixtures
  </action>
  <verify>Install and verify: `pip install -e ".[dev]" && bandit --version`. Should show bandit version 1.7.0+. Verify config loads: `bandit -c .bandit --help` should not error.</verify>
  <done>Bandit is installed as dev dependency, configuration file exists at project root, ready to run security scans.</done>
</task>
