Update the ocm-downloader stage in container/Containerfile to download from the correct repository:

**Current (WRONG):**

**Fix to (CORRECT):**
- Repository: `https://github.com/openshift-online/ocm-cli`
- Tool: OpenShift Cluster Manager CLI (what we actually need)
- Binary filename pattern: `ocm-linux-amd64` (no version in filename for ocm-cli)
- Alternative download source documented: https://console.redhat.com/openshift/downloads#tool-coreos-installer

**Specific changes in ocm-downloader stage (lines 29-37):**

Replace:
```dockerfile
RUN set -e && \
    cd /tmp && \
    curl -fsSL "https://github.com/open-component-model/ocm/releases/download/v${OCM_VERSION}/ocm-${OCM_VERSION}-linux-amd64.tar.gz" -o ocm.tar.gz && \
    curl -fsSL "https://github.com/open-component-model/ocm/releases/download/v${OCM_VERSION}/ocm-${OCM_VERSION}-linux-amd64.tar.gz.sha256" -o expected.sha256 && \
    echo "$(cat expected.sha256)  ocm.tar.gz" | sha256sum --check && \
    tar -xzf ocm.tar.gz && \
    chmod +x ocm && \
    mv ocm /usr/local/bin/ocm && \
    rm -rf /tmp/*
```

With:
```dockerfile
RUN set -e && \
    cd /tmp && \
    curl -fsSL "https://github.com/openshift-online/ocm-cli/releases/download/v${OCM_VERSION}/ocm-linux-amd64" -o ocm && \
    curl -fsSL "https://github.com/openshift-online/ocm-cli/releases/download/v${OCM_VERSION}/ocm-linux-amd64.sha256" -o expected.sha256 && \
    echo "$(cat expected.sha256)  ocm" | sha256sum --check && \
    chmod +x ocm && \
    mv ocm /usr/local/bin/ocm && \
    rm -rf /tmp/*
```

**Key differences:**
1. Repository URL: `openshift-online/ocm-cli` instead of `open-component-model/ocm`
2. Binary format: Direct binary download (`ocm-linux-amd64`) instead of tarball (`ocm-${OCM_VERSION}-linux-amd64.tar.gz`)
3. No tar extraction needed: ocm-cli releases provide direct binary
4. Checksum file: Still `.sha256` format but for direct binary
5. Same fail-fast pattern: `set -e` with `sha256sum --check`
