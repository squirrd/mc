Purpose: Establish the architectural foundation for efficient container builds. Output: Multi-stage Containerfile with three named stages (ocm-downloader, mc-builder, final), automated cache verification test, and validated size reduction with working OCM binary integration.

Convert the existing single-stage Containerfile to a multi-stage build with three named stages:

**Stage 1: ocm-downloader (UBI-minimal base)**
- Use `FROM registry.access.redhat.com/ubi10/ubi-minimal:latest AS ocm-downloader`
- Install curl first (before ARG declaration to maximize caching)
- Declare `ARG OCM_VERSION=0.35.0` after curl install

- Clean up temporary files in same layer
- GitHub URL pattern: `https://github.com/open-component-model/ocm/releases/download/v${OCM_VERSION}/ocm-${OCM_VERSION}-linux-amd64.tar.gz`

**Stage 2: mc-builder (UBI 10.1 base)**
- Use `FROM registry.access.redhat.com/ubi10/ubi:10.1 AS mc-builder`
- Install build dependencies: python3-pip, gcc, python3-devel (chain with dnf clean all and rm -rf /var/cache/dnf in single RUN)
- Install Python build tools: setuptools, wheel (separate RUN with --no-cache-dir)
- Copy project source to /opt/mc
- Install MC CLI in editable mode: `pip3 install --no-cache-dir --no-build-isolation -e /opt/mc`

**Stage 3: final (UBI 10.1 base)**
- Use `FROM registry.access.redhat.com/ubi10/ubi:10.1 AS final`
- Preserve existing LABEL directives (maintainer, description, version)
- Install ONLY runtime dependencies: python3, vim, nano, wget, openssl (chain with dnf clean all in single RUN)
- Create non-root mcuser (preserve existing groupadd/useradd pattern)
- Create /case workspace with correct ownership

<task type="auto">
  <name>Task 2: Create automated cache verification test</name>
  <files>container/verify-cache.sh</files>
  <action>
Create executable shell script that automates layer caching verification:

**Script requirements:**
- Set bash strict mode: `set -euo pipefail`
- Build image twice and capture output to temp files
- Parse second build output for "Using cache" or "CACHED" strings (podman uses "CACHED")
- Verify all stages show cache hits on second build
- Report success/failure with clear messaging
- Clean up temp files on exit (use trap for cleanup)
- Exit 0 on success, exit 1 on failure

**Implementation approach:**
```bash
#!/bin/bash
set -euo pipefail

# Build first time
echo "Building container image (first build)..."
podman build -t mc-rhel10:cache-test -f container/Containerfile . > build1.log 2>&1

# Build second time (no changes)
echo "Building container image (second build, no changes)..."
podman build -t mc-rhel10:cache-test -f container/Containerfile . > build2.log 2>&1

# Check for cache usage in second build
if grep -q -E "(Using cache|CACHED)" build2.log; then
    echo "SUCCESS: Layer caching working - second build used cached layers"
    CACHE_LINES=$(grep -c -E "(Using cache|CACHED)" build2.log)
    echo "Cache hits: ${CACHE_LINES} layers"
else
    echo "ERROR: No cached layers found in second build"
    echo "This indicates cache invalidation - check ARG placement and layer ordering"
    exit 1
fi

# Cleanup
rm -f build1.log build2.log

exit 0
```

**Make script executable:** chmod +x container/verify-cache.sh
