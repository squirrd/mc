<task type="auto">
  <name>Create custom exception hierarchy</name>
  <files>src/mc/exceptions.py</files>
  <action>
Create new file `src/mc/exceptions.py` with custom exception hierarchy following research patterns:

**Base exception:**
- MCError(Exception) - Base for all MC CLI errors
  - exit_code = 1 (default)
  - __init__(message, suggestion=None) - Store message and optional suggestion

**Domain-specific exceptions (inherit from MCError):**
- AuthenticationError - Authentication failures (missing token, expired, permission denied)
  - exit_code = 65 (EX_DATAERR from sysexits.h)
- APIError - Base for API-related errors
  - exit_code = 69 (EX_UNAVAILABLE)
  - Subclass: HTTPAPIError with classmethod from_response(response)
  - Subclass: APITimeoutError
  - Subclass: APIConnectionError
- ValidationError - Input validation failures (invalid case number, bad args)
  - exit_code = 2 (command line syntax error)
- WorkspaceError - Workspace operation failures (can't create workspace)
  - exit_code = 73 (EX_CANTCREAT)
- FileOperationError - File operation failures (permission denied, disk full)
  - exit_code = 74 (EX_IOERR)

**HTTPAPIError.from_response() logic:**
- Map status codes to user-friendly messages:
  - 401: "Authentication failed. Try: mc auth login"
  - 403: "Access forbidden. Check: Your account has case access permissions"
  - 404: "Resource not found. Check: Case number is correct"
  - 429: "Rate limited. Try: Wait a moment and retry"
  - 500: "API server error. Try: Retry in a few minutes"
  - 503: "API temporarily unavailable. Try: Retry in a few minutes"
- Store status_code and response as attributes

Follow Python conventions: Docstrings for each exception, type hints if appropriate.
  </action>

  <name>Create error formatting utilities</name>
  <files>src/mc/utils/errors.py</files>
  <action>
Create new file `src/mc/utils/errors.py` with error formatting and display utilities:

**format_error_message(error: Exception) -> str:**
- Convert exception to user-friendly message
- If error has 'suggestion' attribute, append on new line
- Return formatted string

**handle_cli_error(error: Exception, debug: bool = False) -> int:**
- Log error with logging module (logger.error with exc_info=debug)
- Print formatted error message to stderr using print(..., file=sys.stderr)
- If debug=True, print full traceback to stderr
- Return appropriate exit code:
  - If error has 'exit_code' attribute, return it
  - Otherwise return 1 (generic error)

**Import requirements:**
- import sys
- import logging
- from mc.exceptions import MCError

**Logging configuration note:**
Don't configure logging in this module - that's for main.py. Just use logger = logging.getLogger(__name__).

Follow clig.dev guidelines: Errors to stderr, include suggestions, respect debug flag.
  </action>
