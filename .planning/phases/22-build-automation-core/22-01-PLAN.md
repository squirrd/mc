Automate container builds with version extraction, validation, and orchestration to eliminate manual version management and enable consistent, reproducible builds with dry-run preview capability.

Output: build-container.sh script that reads versions.yaml, validates all versions, builds container with proper --build-arg injection, tags with both semantic version and :latest, supports dry-run mode, and provides clear feedback on success/failure.

  <action>
Create executable bash script at container/build-container.sh implementing core build automation infrastructure:

**Script header:**
- Shebang: #!/usr/bin/env bash
- Error handling: set -euo pipefail (fail-fast on errors, undefined vars, pipeline failures)
- Brief description comment block

**Argument parsing (manual while-loop pattern from RESEARCH.md Pattern 1):**
- Initialize flags: DRY_RUN=false, VERBOSE=false
- Parse with while loop and case statement:
  - --dry-run: Sets DRY_RUN=true
  - --verbose: Sets VERBOSE=true
  - --help: Shows usage and exits 0
  - Unknown option: Error and exit 1
- No positional arguments needed

**Usage/help function (from RESEARCH.md):**
- show_usage() function with heredoc
- Document all flags (--dry-run, --verbose, --help)
- Show examples: normal build, dry-run, verbose
- List requirements: podman, yq (mikefarah/yq), versions.yaml
- Describe output: versioned tag + :latest tag

**Preflight validation (RESEARCH.md Pattern 3, complete validation function):**
- preflight_checks() function that validates:
  - yq installed (command -v yq)
  - yq is correct version (mikefarah/yq, not Python version) - check yq --version output
  - podman installed (command -v podman)
  - podman machine running on macOS (podman machine inspect --format '{{.State}}')
  - container/versions.yaml file exists
- Collect all failures before exiting (don't exit on first error)
- Print actionable error messages (what's wrong + how to fix)
- Exit 1 if any check failed

**Version extraction (RESEARCH.md Pattern 4):**
- extract_and_validate_versions() function
- Use yq to extract:
  - IMAGE_VERSION from .image.version
  - MC_VERSION from .mc.version
  - All tool versions: iterate through .tools keys, extract .tools.{name}.version
- Store tool versions in associative array: declare -A TOOL_VERSIONS
- Example: TOOL_VERSIONS["ocm"]=$(yq '.tools.ocm.version' versions.yaml)

**Version validation (RESEARCH.md Pattern 9):**
- SEMVER_REGEX='^[0-9]+\.[0-9]+\.[0-9]+$'
- Check each extracted version is not empty, not "null", and matches regex
- Fail immediately with clear error if any version invalid
- Error format: "Error: Invalid {context} version format: {value} (expected x.y.z)"

**Architecture detection (RESEARCH.md Pattern 6 - architecture mapping):**
- ARCH=$(uname -m)
- Map to standardized names:
  - x86_64 → amd64
  - aarch64 → arm64
  - Other → Error (unsupported)
- Store in ARCH variable for future use (Phase 25 will use for multi-arch)
- For Phase 22: Just detect and store, only support amd64 (validation check)

**Main execution flow:**
- Call preflight_checks
- Call extract_and_validate_versions
- Detect architecture
- (Task 2 will add build orchestration here)

**Important implementation notes:**
- Always quote variable expansions: "$VAR" not $VAR
- Use command -v not which for command detection
- yq queries use dot notation: '.image.version'
- Check for both empty and "null" strings after yq extraction
- macOS-specific podman machine check only when uname == "Darwin"
- CI-friendly output: no ANSI colors, no special characters, predictable format

<task type="auto">
  <name>Task 2: Add build orchestration with podman execution and feedback</name>
  <files>container/build-container.sh</files>
  <action>
Extend build-container.sh with build orchestration, command execution, and user feedback:

**Build argument construction (RESEARCH.md Pattern 4 + Pattern 7):**
- Create build_args array: build_args=()
- Add MC_VERSION build arg: build_args+=("--build-arg" "MC_VERSION=$MC_VERSION")
- For each tool in TOOL_VERSIONS associative array:
  - Convert tool name to uppercase: ocm → OCM
  - Create ARG name: OCM_VERSION
  - Add to array: build_args+=("--build-arg" "OCM_VERSION=${TOOL_VERSIONS[ocm]}")

**Build command construction (RESEARCH.md Pattern 7 + Pattern 8):**
- perform_build() function
- Define image tags:
  - IMAGE_TAG="mc-rhel10:${IMAGE_VERSION}"
  - LATEST_TAG="mc-rhel10:latest"
- Build command array (RESEARCH.md Pattern 7 - multi-tag):
  ```
  build_cmd=(
    podman build
    --file container/Containerfile
    --tag "$IMAGE_TAG"
    --tag "$LATEST_TAG"
  )
  ```
- Append build args: build_cmd+=("${build_args[@]}")
- Add context path: build_cmd+=(.)
- Control verbosity: if not VERBOSE, add --quiet flag

**Dry-run implementation (RESEARCH.md Pattern 5):**
- If DRY_RUN=true:
  - Echo "[DRY-RUN] Would execute:"
  - Print full command: printf '%s ' "${build_cmd[@]}" + newline
  - Show parsed versions:
    - Image version: IMAGE_VERSION
    - MC CLI version: MC_VERSION
    - Tool versions: iterate TOOL_VERSIONS and print each
  - Show tags that would be created: IMAGE_TAG and LATEST_TAG
  - Exit 0 (successful validation)
- Make dry-run output visually distinct (clear DRY-RUN prefix on each line)

**Build execution:**
- If not dry-run, execute: "${build_cmd[@]}"
- Podman output shown based on VERBOSE flag (--quiet suppresses unless VERBOSE)
- Exit code from podman build propagates (set -e handles failure)

**Progress feedback (22-CONTEXT.md decisions):**
- Echo major steps:
  - "Reading versions.yaml..."
  - "Starting build..." (show IMAGE_VERSION and MC_VERSION before podman runs)
  - "Build completed successfully" (after podman succeeds)
- Success message includes:
  - Image tags created: IMAGE_TAG and LATEST_TAG
  - Build time: Use SECONDS builtin (RESEARCH.md Pattern 6)
- Error messages:
  - On podman failure: "Build failed. Re-run with --verbose for details"
  - Include exit code from podman

**Build time tracking (RESEARCH.md Pattern 6):**
- Reset SECONDS=0 at start of perform_build
- After build completes: ELAPSED=$SECONDS
- Show in success message: "Build completed in ${ELAPSED}s"

**Main execution flow (complete script):**
- Parse arguments (while loop)
- Call preflight_checks
- Echo "Reading versions.yaml..."
- Call extract_and_validate_versions
- Detect architecture
- Validate architecture is amd64 (for Phase 22 - error if not)
- Echo "Starting build..."
- Echo "  Image version: $IMAGE_VERSION"
- Echo "  MC CLI version: $MC_VERSION"
- Call perform_build
- (If not dry-run and successful) Echo success message with tags and time

**Important implementation notes:**
- Use array for build command, not string (proper quoting, handles spaces)
- Dry-run must perform ALL validation (reliable preview)
- Dry-run exits 0 on success (matches user expectation per RESEARCH.md)
- Progress output goes to stdout, errors to stderr (>&2)
- Keep default output brief and CI-friendly (22-CONTEXT.md)
- Verbose flag shows podman build output, normal mode suppresses with --quiet
- SECONDS is automatic bash builtin, no initialization needed beyond reset
