<task type="auto">
  <name>Task 1: Upgrade Python version and configure mypy strict mode</name>
  <files>pyproject.toml</files>
  <action>
Update Python version requirement and mypy configuration:

1. Change `requires-python = ">=3.8"` to `requires-python = ">=3.11"`
   - Rationale: Python 3.11 enables modern union syntax (X | Y) without __future__ imports, improved error messages, and better performance (10-60% faster than 3.10)
   - 3.11 is stable (released Oct 2022, current 3.11.x is well-tested)
   - Avoids 3.10 which requires `from __future__ import annotations` for modern syntax

2. Update classifiers to remove Python 3.8, 3.9, 3.10 (keep 3.11 and add 3.12, 3.13)
3. Update black target-version to `['py311', 'py312', 'py313']`
4. Add types-requests to dev dependencies: `"types-requests>=2.32.0"`
5. Update mypy configuration:
   ```toml
   [tool.mypy]
   python_version = "3.11"
   warn_return_any = true
   warn_unused_configs = true
   check_untyped_defs = true
   disallow_untyped_defs = true
   disallow_incomplete_defs = true
   disallow_untyped_decorators = false
   no_implicit_optional = true
   warn_redundant_casts = true
   warn_unused_ignores = true
   warn_no_return = true

   [[tool.mypy.overrides]]
   module = "tqdm.*"
   ignore_missing_imports = true

   [[tool.mypy.overrides]]
   module = "platformdirs.*"
   ignore_missing_imports = true
   ```
6. Remove `tomli` from dependencies since Python 3.11+ has stdlib tomllib
7. Keep tomli-w for writing TOML (no stdlib writer)

Why these settings:
- check_untyped_defs enables checking function bodies even before all signatures have types
- disallow_untyped_defs requires all function signatures to have types (strict goal)
- no_implicit_optional prevents Optional[X] when default is None (explicit is better)
- Per-module overrides for tqdm/platformdirs which may lack complete stubs
  </action>

Add comprehensive type hints to all functions following these principles:

**Type Syntax (Python 3.11+):**
- Use `X | Y` instead of `Union[X, Y]`
- Use `X | None` instead of `Optional[X]`
- Use `list[str]`, `dict[str, Any]` instead of `List[str]`, `Dict[str, Any]`
- Use `tuple[float, float]` for tuples
- Remove `from typing import Optional, Union, List, Dict` imports
- Keep `from typing import Any, TypedDict, Literal` imports where needed

**For integrations/redhat_api.py:**
- Add TypedDict for API responses:
  ```python
  from typing import TypedDict, NotRequired, Any

  class CaseDetails(TypedDict):
      summary: str
      accountNumberRef: str
      status: str
      severity: NotRequired[str]
      product: NotRequired[str]

  class AttachmentMetadata(TypedDict):
      fileName: str
      link: str
      fileSize: NotRequired[int]
  ```
- Update method signatures:
  - `fetch_case_details(self, case_number: str) -> CaseDetails:`
  - `fetch_account_details(self, account_number: str) -> dict[str, Any]:`
  - `list_attachments(self, case_number: str) -> list[AttachmentMetadata]:`
  - `download_file(self, url: str, local_filename: str, force: bool = False) -> None:`
  - `get_ca_bundle() -> str | bool:` (returns path or True)
  - `check_download_safety(...) -> tuple[bool, str | None]:`

**For cli/main.py:**
- Add exit code type:
  ```python
  from typing import Literal

  ExitCode = Literal[0, 1, 2, 65, 69, 73, 74, 130]

  def main() -> ExitCode:
  ```

- formatters.py: `shorten_and_format(input_string: str) -> str:`
- validation.py: Already typed, modernize to `X | None` syntax

**For cli/commands/:**
