class SalesforceAPIClient:
    """Client for Red Hat Salesforce case metadata queries."""

    def __init__(self, username: str, password: str, security_token: str)
    def _create_session(self) -> Salesforce  # Track token expiry (2 hours)
    def _needs_refresh(self) -> bool  # Check if within 5 min of expiry
    def _refresh_session(self) -> None  # Proactive token refresh

    @retry(stop=stop_after_attempt(3), wait=wait_exponential(multiplier=1, min=2, max=8))
    def query_case(self, case_number: str) -> dict[str, Any]  # SOQL query for case metadata

**Implementation requirements:**
- Use `simple-salesforce` library (NOT requests - library handles session management)
- Token tracking: Store `_token_expires_at = time.time() + 7200` on session creation (2 hour Salesforce token lifetime)

- SOQL query fields: CaseNumber, Account.Name, Subject, Severity__c, Status, Owner.Name, CreatedDate
- Respect Retry-After header on HTTP 429 (rate limiting) - parse header and wait before retry
- Handle 401/403 as permanent failures (fail fast, don't retry auth errors)
- Follow RedHatAPIClient error handling pattern: HTTPAPIError with actionable suggestions

```python
def get_config_mount_spec(config_path: Path) -> dict[str, str]:
    """Generate Podman mount spec for read-only config access.

    Returns dict with:
        - source: Host config file path
        - target: Container config file path (/root/.mc/config.toml)
        - options: "ro" (read-only)
    """
    if not config_path.exists():
        raise ConfigError(f"Config file not found: {config_path}")

    return {
        "source": str(config_path.resolve()),
        "target": "/root/.mc/config.toml",
        "options": "ro"  # Read-only - containers can read but not modify
    }
```

**WHY these choices:**
- Keep Salesforce credentials separate from Red Hat API section (different auth mechanisms)
- Validate all three OAuth2 fields (username, password, security_token) - Salesforce requires all three
- Empty strings as defaults (user must populate via config wizard)
- Read-only mount prevents containers from corrupting host config
- Mount to /root/.mc/config.toml (standard container config location)
- Per CONTEXT.md: Mount host config directory, tokens refreshed by host process
