<objective>
Create the `mc-update` command as a standalone entry point that lets users upgrade MC CLI via `uv tool upgrade mc`, validates the result, and prints actionable recovery instructions on failure.

Purpose: Users need a single command that safely upgrades MC CLI with clear feedback. The separate entry point design ensures `mc-update` itself survives the package replacement during upgrade.

Output: `src/mc/update.py` (the module), `pyproject.toml` updated with the entry point, and `tests/unit/test_update.py` covering all primary flows.
</objective>

  <name>Task 1: Create src/mc/update.py with full upgrade logic</name>
  <files>src/mc/update.py</files>
  <action>
Create `src/mc/update.py` as a self-contained module. Do NOT import from `mc.cli.main` — mc-update must remain independent so it works even if mc's CLI machinery is broken during a partial upgrade.

Module structure:
- Module-level docstring explaining the mc-update entry point purpose
- `import argparse, logging, subprocess, sys` at top
- `from typing import Literal` — use `ExitCode = Literal[0, 1]` as the return type for `upgrade()`
- `logger = logging.getLogger(__name__)`

Function `_run_upgrade() -> int`:
- Runs `subprocess.run(['uv', 'tool', 'upgrade', 'mc'], capture_output=False, text=True, check=False)`
- Use `capture_output=False` so uv's live progress output streams directly to the terminal
- Wraps the call in `try/except FileNotFoundError` — on FileNotFoundError, print to stderr: `"Error: uv not found. Install from https://docs.astral.sh/uv/"` and return 1
- Returns `result.returncode` on success path

Function `_verify_mc_version() -> bool`:
- Runs `subprocess.run(['mc', '--version'], capture_output=True, text=True, check=False)`
- Use `capture_output=True` here (we need to inspect and print the output, not stream it)
- Wraps in `try/except FileNotFoundError` — on FileNotFoundError, print to stderr: `"Error: mc not found on PATH after upgrade. Check: uv tool list"` and return False
- On returncode == 0: print `result.stdout.strip()` to stdout and return True
- On returncode != 0: return False (caller prints recovery instructions)

Function `_print_recovery_instructions() -> None`:
- Prints three lines to stderr:
  - `""` (blank line for visual separation)
  - `"Upgrade failed. To recover, run:"`
  - `"  uv tool install --force mc"`

Function `upgrade() -> ExitCode`:
- First line: `from mc.runtime import is_agent_mode` (lazy import inside function body — keeps mc-update independent at import time, pulls in runtime only when executing)
- If `is_agent_mode()`: print to stderr `"mc-update is not available in agent mode. Run mc-update on the host."` and return 1
- Print `"Upgrading MC CLI..."` to stdout
- Call `_run_upgrade()`. If returncode != 0: call `_print_recovery_instructions()` and return 1
- Print `"Verifying upgrade..."` to stdout
- Call `_verify_mc_version()`. If False: call `_print_recovery_instructions()` and return 1
- Print `"Upgrade complete."` to stdout and return 0

Function `main() -> None`:
- Creates `argparse.ArgumentParser(prog='mc-update', description='MC CLI updater')`
- Adds subparsers group with `dest='command'`
- Adds `upgrade` subparser: `subparsers.add_parser('upgrade', help='Upgrade MC CLI via uv tool upgrade')`
- Parses args with `parser.parse_args()`
- If `args.command == 'upgrade'`: `sys.exit(upgrade())`
- Else: `parser.print_help()` and `sys.exit(0)`
- Guard at bottom: `if __name__ == '__main__': main()`

All functions must have complete type hints. All functions must have docstrings. Run `uv run mypy src/mc/update.py` to verify type checking passes with no errors.

Test classes and cases to implement:

**class TestRunUpgrade:**
- `test_run_upgrade_success`: patch `mc.update.subprocess.run` to return `MagicMock(returncode=0)`. Assert `_run_upgrade()` returns 0.
- `test_run_upgrade_nonzero_exit`: patch `mc.update.subprocess.run` to return `MagicMock(returncode=1)`. Assert `_run_upgrade()` returns 1.
- `test_run_upgrade_uv_not_found`: patch `mc.update.subprocess.run` to raise `FileNotFoundError`. Assert `_run_upgrade()` returns 1 and `capsys.readouterr().err` contains `"uv not found"`.

**class TestVerifyMcVersion:**

**class TestUpgrade:**

- `test_upgrade_success`: `monkeypatch.delenv("MC_RUNTIME_MODE", raising=False)`. Patch `mc.update.subprocess.run` with `side_effect` list: first call returns `MagicMock(returncode=0)` (upgrade), second returns `MagicMock(returncode=0, stdout="mc 2.0.5\n")` (version check).
