<task type="auto">
  <name>Task 2: Validate manual build workflow with version injection</name>
  <files>None (verification task demonstrating manual workflow)</files>
  <action>
Demonstrate that versions.yaml can be parsed and values injected into Containerfile build process. This establishes the pattern that Phase 22 build automation will implement.

**Manual workflow steps:**

1. Extract OCM version from versions.yaml using yq:
```bash
cd /Users/dsquirre/Repos/mc
OCM_VERSION=$(yq '.tools.ocm.version' container/versions.yaml)
echo "Extracted OCM version: $OCM_VERSION"
```

2. Build container image with extracted version injected as ARG:
```bash
podman build \
  --build-arg OCM_VERSION=$OCM_VERSION \
  -t mc-rhel10:1.0.0 \
  -f container/Containerfile .
```

3. Verify injected version matches versions.yaml:
```bash
# Run container and check OCM version
podman run --rm mc-rhel10:1.0.0 ocm version | grep -o "[0-9]\+\.[0-9]\+\.[0-9]\+" > /tmp/ocm_version.txt
ACTUAL_VERSION=$(cat /tmp/ocm_version.txt)
EXPECTED_VERSION=$(yq '.tools.ocm.version' container/versions.yaml)

if [ "$ACTUAL_VERSION" = "$EXPECTED_VERSION" ]; then
  echo "SUCCESS: OCM version in container ($ACTUAL_VERSION) matches versions.yaml ($EXPECTED_VERSION)"
else
  echo "FAILED: Version mismatch - container has $ACTUAL_VERSION, versions.yaml has $EXPECTED_VERSION"
  exit 1
fi
```

4. Test with different version (demonstrate flexibility):
```bash
# Extract different version (e.g., 1.0.8 from Phase 20 testing)
podman build \
  --build-arg OCM_VERSION=1.0.8 \
  -t mc-rhel10:test \
  -f container/Containerfile .

# Verify different version was injected
podman run --rm mc-rhel10:test ocm version | grep "1.0.8" && echo "Version override works"
```

**Success criteria:**
- yq successfully extracts version from YAML
- podman build accepts extracted version via --build-arg
- Container OCM binary version matches versions.yaml version
- Different versions can be injected (demonstrates ARG injection works)

**What this proves:**
- versions.yaml is valid single source of truth
- Manual extraction and injection workflow is viable
- Phase 22 can automate this exact pattern
- ARG injection architecture from Phase 20 works with versions.yaml
