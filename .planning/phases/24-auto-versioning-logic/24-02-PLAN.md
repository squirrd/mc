<objective>
Fix minor version regex validation to reject leading zeros, bringing it into compliance with semantic versioning 2.0.0 specification. The current regex `^[0-9]+\.[0-9]+$` incorrectly accepts invalid versions like "01.2" and "1.02" which violate semver rules.

Purpose: Ensure consistent semver validation between full version (x.y.z) and minor version (x.y) formats. Prevents invalid version numbers from being stored in versions.yaml.

Output: Updated MINOR_VERSION_REGEX that matches the semver 2.0.0 pattern already used for full version validation.
</objective>

  <action>
Update the MINOR_VERSION_REGEX constant to enforce semver 2.0.0 compliance by rejecting leading zeros:

**Current regex (line 23):**
```bash
MINOR_VERSION_REGEX='^[0-9]+\.[0-9]+$'
```

**Updated regex:**
```bash
MINOR_VERSION_REGEX='^(0|[1-9][0-9]*)\.(0|[1-9][0-9]*)$'
```

**What this fixes:**
- REJECTS: "01.2" (leading zero in major)
- REJECTS: "1.02" (leading zero in minor)
- REJECTS: "00.0" (leading zero in major)
- ACCEPTS: "0.0" (zero is valid)
- ACCEPTS: "1.2" (standard version)
- ACCEPTS: "10.5" (multi-digit without leading zeros)

**Pattern explanation:**
- `(0|[1-9][0-9]*)` matches either "0" OR a non-zero digit followed by any digits
- This is identical to the pattern already used in SEMVER_REGEX for full version validation
- Ensures consistency: both full (x.y.z) and minor (x.y) validation follow the same rules

**Rationale:**
Semantic Versioning 2.0.0 specification explicitly states: "A normal version number MUST take the form X.Y.Z where X, Y, and Z are non-negative integers, and MUST NOT contain leading zeroes." This regex enforces that requirement for the minor version format stored in versions.yaml.

**What NOT to change:**
- Don't modify SEMVER_REGEX (already correct)
- Don't modify validation logic or error messages
- Don't modify where MINOR_VERSION_REGEX is used (line 183)
- Only update the regex pattern definition at line 23
  </action>
