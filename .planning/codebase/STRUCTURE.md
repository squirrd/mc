## Directory Purposes

**src/mc:**
- Purpose: Main application source code
- Contains: All Python modules for the application
- Key files: `version.py` contains version and description

**src/mc/cli:**
- Purpose: Command-line interface implementation
- Contains: Main entry point and command handlers
- Key files:
  - `main.py`: Entry point with argument parsing and routing
  - `commands/__init__.py`: Empty module marker
  - `commands/case.py`: Case management commands (attach, check, create, case-comments)
  - `commands/other.py`: Utility commands (ls, go)

**src/mc/controller:**
- Purpose: Business logic and domain models
- Contains: Workspace management
- Key files:
  - `workspace.py`: WorkspaceManager class for case workspace operations

**src/mc/integrations:**
- Purpose: External system integrations
- Contains: API clients and external service interfaces
- Key files:
  - `redhat_api.py`: RedHatAPIClient for Red Hat Support API
  - `ldap.py`: LDAP search functionality

**src/mc/utils:**
- Purpose: Shared utility functions
- Contains: Helper functions used across the application
- Key files:
  - `auth.py`: OAuth token management
  - `file_ops.py`: File system operations
  - `formatters.py`: String formatting utilities

**src/mc/config:**
- Purpose: Configuration management
- Contains: Empty placeholder modules
- Key files: `__init__.py`, `schemas/__init__.py`

**src/mc/agent:**
- Purpose: Future agent functionality
- Contains: Empty placeholder module
- Key files: `__init__.py`

**tests:**
- Purpose: Test suite for the application
- Contains: Unit tests, integration tests, fixtures
- Key files:
  - `test_*.sh`: Bash-based integration test scripts
  - `unit/__init__.py`: Unit tests directory (empty)
  - `integration/__init__.py`: Integration tests directory (empty)
  - `fixtures/test_data/`: Test data files

**config-examples:**
- Purpose: Example configuration files for users
- Contains: YAML configuration templates
- Key files:
  - `case.yaml.example`
  - `compact.yaml.example`
  - `customer.yaml.example`
  - `mounts.yaml.example`

**container:**
- Purpose: Container build and runtime files
- Contains: Containerfile and scripts
- Key files:
  - `Containerfile`: Container image definition
  - `build.sh`: Container build script
  - `run.sh`: Container run script

**docs:**
- Purpose: Project documentation
- Contains: Markdown documentation files
- Key files: `RESTRUCTURE_SUMMARY.md`

**archive:**
- Purpose: Legacy code and old implementations
- Contains: Previous versions and deprecated code
- Key files: Various old scripts and prototypes

**.planning:**
- Purpose: GSD planning and analysis documents
- Contains: Codebase analysis, architecture docs
- Key files: Generated by GSD commands

**.claude:**
- Purpose: Claude AI agent configurations
- Contains: Agent definitions, hooks, commands
- Key files: Custom GSD workflow definitions

**scripts:**
- Purpose: Utility scripts
- Contains: Currently empty

## Naming Conventions

**Files:**
- Python modules: `lowercase_with_underscores.py`
- Shell scripts: `lowercase_with_underscores.sh` or `kebab-case.sh`
- Config examples: `name.yaml.example`
- Test scripts: `test_name.sh`

**Directories:**
- Package directories: `lowercase` (e.g., `cli`, `utils`, `controller`)
- Test directories: `lowercase` (e.g., `unit`, `integration`, `fixtures`)
- Documentation: `lowercase` or `kebab-case` (e.g., `docs`, `config-examples`)

**Python Identifiers:**
- Functions: `lowercase_with_underscores()` (e.g., `get_access_token()`)
- Classes: `PascalCase` (e.g., `WorkspaceManager`, `RedHatAPIClient`)
- Variables: `lowercase_with_underscores` (e.g., `case_number`, `base_dir`)
- Constants: No explicit constants pattern observed

**Module Organization:**
- Each package has `__init__.py` (mostly empty)
- Related functionality grouped in modules (e.g., all case commands in `case.py`)
- No barrel exports or re-exports observed

## Where to Add New Code

**New Case Command:**
- Primary code: `src/mc/cli/commands/case.py` (add new function)
- CLI registration: `src/mc/cli/main.py` (add subparser and routing)
- Tests: `tests/test_<command_name>.sh`

**New Utility Command:**
- Primary code: `src/mc/cli/commands/other.py` (add new function)
- CLI registration: `src/mc/cli/main.py` (add subparser and routing)
- Tests: `tests/test_<command_name>.sh`

**New Integration:**
- Implementation: `src/mc/integrations/<service_name>.py`
- Client class following pattern of `RedHatAPIClient`
- Usage: Import in command modules

**New Domain Logic:**
- Implementation: `src/mc/controller/<domain>.py`
- Class-based following pattern of `WorkspaceManager`
- Usage: Import in command layer

**Utilities:**
- Shared helpers: `src/mc/utils/<category>.py`
- Group related functions in single module (e.g., all auth in `auth.py`)

**Configuration:**
- Schema definitions: `src/mc/config/schemas/`
- Config loading: `src/mc/config/`
- Examples: `config-examples/<name>.yaml.example`

**Tests:**
- Bash integration tests: `tests/test_<feature>.sh`
- Unit tests: `tests/unit/test_<module>.py` (when unit testing is set up)
- Integration tests: `tests/integration/test_<feature>.py` (when set up)
- Test data: `tests/fixtures/test_data/`
