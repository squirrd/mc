# Milestone v2.0.2: Window Tracking

**Status:** ✅ SHIPPED 2026-02-08
**Phases:** 15-19
**Total Plans:** 10

## Overview

Add window ID tracking to MC CLI's terminal automation to eliminate duplicate terminal windows. The system extends existing SQLite infrastructure with a window registry, captures platform-specific window IDs at creation time, and focuses existing windows instead of creating duplicates. Five phases deliver incrementally: foundation registry, macOS implementation, cleanup mechanisms, Linux support, and comprehensive testing.

## Milestone Summary

**Key Decisions:**
- Window ID tracking chosen over title-based search (titles volatile in iTerm2)
- Extend existing StateDatabase with window_registry table (zero new dependencies)
- AppleScript window `id` property for macOS, wmctrl for Linux X11
- Lazy validation with auto-cleanup (self-healing registry)
- First-write-wins via sqlite3.IntegrityError handling
- 0.5s delay after launch before capturing window ID (terminal initialization time)
- Callback-based validator in lookup() for platform-agnostic window validation

**Issues Resolved:**
- Duplicate terminal windows on repeated `mc case XXXXX` calls
- Unreliable title-based window search (iTerm2 title changes)
- No cleanup mechanism for stale window entries
- test_duplicate_terminal_prevention_regression integration test failing

**Technical Achievements:**
- SQLite-backed window registry with WAL mode for concurrent multi-process access
- Cross-platform window tracking (macOS + Linux X11)
- Self-healing registry with automatic stale entry cleanup
- 530 tests passing with 74.65% coverage
- Graceful database corruption recovery
- Desktop-native terminal preference on Linux (konsole on KDE, gnome-terminal on GNOME)
