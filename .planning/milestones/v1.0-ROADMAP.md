# Milestone v1.0: Hardening

**Status:** ✅ SHIPPED 2026-01-22
**Phases:** 1-8
**Total Plans:** 21

## Overview

Transform the MC CLI from a working prototype into a production-ready tool by systematically addressing technical debt, bugs, security issues, and performance problems. The journey follows a dependency-driven approach: establish test infrastructure first (foundation for all other work), then tackle configuration and error handling (unblock testing and improve UX), followed by logging and performance optimizations (build on solid foundation). Each phase delivers a coherent, verifiable capability while maintaining backward compatibility.

## Milestone Summary

**Key Decisions:**

- Test infrastructure before fixes (TDD approach establishes testing foundation first)
- Modern pytest with importlib mode (use pytest 9.0+ for better namespace handling)
- HTTP mocking via responses library (instead of generic pytest-mock for requests)
- TOML chosen for config file format (Python 3.11+ stdlib support)
- File-based token cache over keyring library (simpler, no dependencies)
- Python 3.11 minimum version (native union syntax without __future__ imports)
- mypy as separate test command (not pytest plugin for cleaner separation)
- TypedDict for API responses (provides structure without runtime overhead)
- 8 concurrent downloads via ThreadPoolExecutor (balances speed and resource usage)
- Rich progress library for multi-file progress tracking
- Backoff library for exponential retry with jitter

**Issues Resolved:**

- No unit test coverage → 100+ tests with 80%+ coverage on critical modules
- All tests hit live APIs → Full mocking infrastructure with responses library
- Hardcoded base directory → TOML config with cross-platform support
- Performance issues → 8x faster downloads, 30-minute metadata caching
- Missing logging infrastructure → Structured logging with sensitive data redaction
- Fragile parsing → Robust error handling with pathlib migration
- Sequential downloads → Parallel downloads with progress bars
- No type safety → 98% type coverage with mypy strict validation

**Technical Debt Carried Forward:**

- 2 cosmetic type annotation gaps in config module (ConfigManager.__init__ missing -> None, Dict[] vs dict[] syntax)
- 20 test failures from Path vs string type changes (tests need modernization to align with typed APIs)
