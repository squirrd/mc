# Milestone v2.0.5: Auto-Update & Terminal

**Status:** SHIPPED 2026-03-12
**Phases:** 29-32
**Total Plans:** 8

## Overview

MC CLI auto-update functionality and iTerm2 Python API migration for cleaner terminal management. Users can now explicitly upgrade MC, pin versions, and see update notifications at startup. macOS terminal windows open via iTerm2's Python API with MCC-Term profile and the raw podman exec command hidden from scrollback.

## Phases

## Milestone Summary

**Total Duration:** ~40 min across 8 plans (same day: 2026-03-12)

**Key Decisions:**

- iterm2 as optional `[macos]` extra — Keeps Linux installs unaffected; runtime ImportError guard for real gate
- asyncio.timeout(5) inside coroutine — run_until_complete() is synchronous; timeout must live inside async def
- `_last_api_window_id` instance attribute — launch() returns None (protocol); window ID threaded through instance
- mc-update independent from mc.cli.main — Survives partial package replacement during upgrade
- Lazy imports inside function bodies — Keeps modules importable even if mc CLI is partially broken
- `pinned_mc='latest'` as no-pin sentinel — Consistent with existing ConfigManager.get_version_config() defaults
- Banner uses daemon thread + threading.Event — 1.5s bounded wait; CLI never blocked beyond timeout
- Calendar-day comparison for suppression — `.date()` comparison, not rolling 24h window
- VersionChecker removed from main.py — Banner replaces background check; cleaner single notification path

**Issues Resolved:**

- Pre-existing `.flake8` misconfiguration (79-char limit vs Black's 100-char) — fixed in Phase 29
- Pre-existing hardcoded `'2.0.1'` version assertion in test_entry_points.py — fixed in Phase 31
- Stale VersionChecker patch in test_main.py after VersionChecker removal — auto-fixed in Phase 32

**Technical Debt Incurred:**

- `_build_iterm_script()` kept as dead code for backwards-compat and test references; cleanup deferred
- Banner agent-mode guard at main.py:151 has no test coverage — suggested fix: 2 unit tests in test_main.py
- Human verification deferred: visual Rich Panel rendering, piped-run suppression, per-day deduplication across real invocations, MCC-Term profile in iTerm2
