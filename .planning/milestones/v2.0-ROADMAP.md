# Milestone v2.0: Containerization + Distribution

**Status:** ✅ SHIPPED 2026-02-01
**Phases:** 9-14.1 (7 phases)
**Total Plans:** 22

## Overview

Transform MC from a traditional CLI tool into a container orchestrator providing isolated per-case workspaces. The roadmap progresses from foundation (Podman integration and platform detection) through core orchestration (container lifecycle and state management) to user-facing features (terminal automation and Salesforce integration) to delivery (container image with tooling and modern distribution).

## Milestone Summary

**Decimal Phases:**

- Phase 14.1: UAT Bug Fixes (inserted after Phase 14 for critical UAT issues discovered during testing)

**Key Decisions:**

- Lazy platform detection — No import-time overhead, testable without Podman
- Proactive Salesforce token refresh — Refresh 5 minutes before 2-hour expiry
- Auto-restart pattern — Stopped containers restart instead of creating duplicates
- Protocol-based launcher abstraction — Enables static type checking with platform flexibility
- uv unified tooling — 10-100x faster than pip/pipx, modern packaging approach
- Backwards compatibility with Phase 14.1-02 — Container commands work without Salesforce

**Issues Resolved:**

- Container orchestration working with full lifecycle management
- Platform differences handled transparently (macOS Podman machine vs Linux)
- Terminal automation across iTerm2, Terminal.app, gnome-terminal, konsole
- RHEL 10 container image built successfully (549 MB with essential tools)
- Modern distribution via uv tool (dev/UAT/prod workflows)
- UAT blocking bugs fixed (auth boundaries, validation, config naming)

**Issues Deferred (v2.0.1):**

- Terminal attachment Salesforce API method call (get_case vs query_case) — **CRITICAL BUG**
- Podman URI byte string errors still occurring despite Phase 14.1-01 fix
- Cache database initialization failures on second mc create run
- Runtime mode detection created but not integrated into decision logic
- Phase 13 missing VERIFICATION.md (deliverables present and functional)

**Technical Debt Incurred:**

- No E2E integration test for full `mc case <number>` workflow (would have caught Salesforce API bug)
- Runtime mode detection module unused (foundation for future features)
- jq tool excluded from base image (not in UBI repos, runtime install via EPEL documented)
