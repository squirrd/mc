# Milestone v2.0.3: Container Tools

**Status:** ✅ SHIPPED 2026-02-10
**Phases:** 20-25
**Total Plans:** 9 (including 2 gap closure plans)

## Overview

Multi-stage container architecture with efficient layer caching and versioned tool management. This milestone established independent image versioning, build automation, and registry integration with intelligent auto-versioning.

### Phase 23: Quay.io Integration

**Goal**: Query registry for latest tags and detect version staleness
**Depends on**: Phase 22
**Plans**: 1 plan

Plans:

- [x] 23-01: Registry query and digest comparison with skopeo integration

**Details:**

Registry query capability via skopeo for OCI standards compliance and automatic credential handling. Four registry functions: query latest version, check version existence, get digest, compare digests. Exponential backoff with jitter (1s→2s→4s→8s→16s) for HTTP 429 rate limiting. Machine-readable JSON output mode (--json) for CI/CD integration. Fail-fast error handling for authentication and network failures.

**Success Criteria Verified:**

2. ✓ Build script detects if local version already exists on quay.io

### Phase 24: Auto-Versioning Logic

**Goal**: Implement intelligent patch version bumping based on digest comparison
**Depends on**: Phase 23
**Plans**: 2 plans

Plans:

- [x] 24-01: Auto-versioning logic with digest-based bumping and auto-push

Registry as source of truth for current patch version. Auto-push new versions when digest differs. Semantic version validation against semver 2.0.0 spec (rejects leading zeros, missing components). Version conflict detection before push.

## Milestone Summary

**Key Decisions:**

- Independent image versioning (1.0.0) decoupled from MC CLI version (2.0.2) enables tool updates without code releases
- Multi-stage architecture with three named stages reduces final image size by excluding build tools
- Registry-as-source-of-truth prevents version conflicts when multiple builds occur
- Digest-based bumping immune to comment/whitespace changes
- Skopeo chosen over direct curl for OCI compliance and automatic credential handling
- Architecture-aware downloads support both amd64 and arm64 platforms
- Pre-flight validation prevents wasted 2-5 minute build cycles on auth failures

**Issues Resolved:**

- Fixed wrong OCM tool integration (open-component-model/ocm → openshift-online/ocm-cli)
- Fixed semver validation regex to reject leading zeros per semver 2.0.0 spec
- Added architecture detection for macOS arm64 (reports "arm64" not "aarch64")
- Fixed OCM version parsing to handle plain version output format

**Issues Deferred:**

None - all requirements satisfied with zero technical debt

**Technical Debt Incurred:**

None - all implementation is production-ready with zero TODOs, FIXMEs, or deferred work items
