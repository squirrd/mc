## Tech Debt (Non-Critical)

No blockers. Items below are deferred — can be tracked in backlog.

### Phase 29 — iTerm2 API Migration

**1. Dead code: `_build_iterm_script()`**
- `src/mc/terminal/macos.py` — iTerm2 AppleScript method retained for backwards compat and test references
- Never called in primary execution path; method docstring marks it as intentionally preserved
- Cleanup candidate: remove in a future milestone once test references updated

### Phase 31 — Version Pinning

**2. Minor dict access inconsistency in `upgrade()`**
- `upgrade()` reads `version_config.get("pinned_mc", "latest")` (defensive `.get()`)
- `pin()`, `unpin()`, `check()` use `version_config["pinned_mc"]` (direct key access)
- `get_version_config()` always includes `pinned_mc` with `"latest"` default, so both are equivalent
- Not a bug; minor stylistic inconsistency only

### Phase 32 — Update Notifications

**3. Banner agent-mode guard untested**
- `main.py:151`: `if get_runtime_mode() != 'agent': show_update_banner()` has no test coverage
- `test_main.py` patches `show_update_banner` away in all test cases — if the guard condition were inverted, no unit test would catch it
- Suggested fix: add `test_show_update_banner_called_when_not_agent` and `test_show_update_banner_suppressed_in_agent_mode` to `test_main.py`

**4. Human verification deferred**
Items that require live iTerm2 session to verify (documented in phase VERIFICATION.md files):
- Visual: MCC-Term profile applied to new window

- Fallback: Terminal.app opens when iTerm2 API is disabled

## Key Decisions Made This Milestone

| Decision | Rationale | Outcome |
|----------|-----------|---------|
| iterm2 as optional `[macos]` extra | Keeps Linux installs unaffected; runtime ImportError guard for real gate | Clean platform separation |
| asyncio.timeout(5) inside coroutine | run_until_complete() is synchronous; timeout must live inside async def | Correct timeout implementation |
| `_last_api_window_id` instance attribute | launch() returns None (protocol); window ID threaded through instance | attach.py consumption works |
| mc-update independent from mc.cli.main | Survives partial package replacement during upgrade | Entry point survives self-upgrade |
| Lazy imports inside function bodies | Keeps modules importable even if mc CLI is partially broken | Resilient during upgrade |
| `pinned_mc='latest'` as no-pin sentinel | Consistent with existing ConfigManager.get_version_config() defaults | Symmetric read/write paths |
| Banner uses daemon thread + threading.Event | 1.5s bounded wait; CLI never blocked beyond timeout | Non-blocking startup behavior |
| Calendar-day comparison for suppression | `.date()` comparison, not rolling 24h window | Simpler; resets at midnight |
| VersionChecker removed from main.py | Banner replaces background check; cleaner single notification path | No duplicate notification logic |
